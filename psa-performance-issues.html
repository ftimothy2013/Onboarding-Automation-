🚨 17% of All Calls Touched a Performance or Access Issue

448 of 2,637 calls contained at least one mention of a login failure, access error, performance problem, or UI defect. Password resets alone appear in 159 calls — they are happening constantly across the client base. The 403 error spike in late February is directly tied to a Redis memory overload event confirmed in internal stand-ups. The Auth service rewrite (PHP → new stack, 50–60% complete as of March 13) is the root cause behind most login and performance complaints.

🔬Root Causes & Recommendations
⚙️
Auth Service PHP Rewrite (50–60% Done)
The primary driver of login failures, 403 errors, and performance issues. Confirmed in Novalera call (Mar 13). Expected completion early April — this is the most critical fix in flight.
🧠
Redis Memory Overload (Feb 24–27 Spike)
Caused the most acute 403 error spike. Clearing cache / incognito was the temporary workaround given to clients. Engineering confirmed fix deployed — but monitoring still active as of Mar 12.
🔗
Linked Billing + PSA Login Conflicts
Users with accounts in both Billing and PSA experience login conflicts. Admins must manually assign roles to new users. Structural fix needed — not just a workaround.
🔑
Password Reset Not Self-Serviceable
159 calls confirm that clients cannot reset their own passwords without admin or SA intervention. This is the #1 volume driver. Front-end self-service reset needs to be unblocked.
💾
Server Migration (Nov 2025)
The November server move caused a 10-hour outage (Xmission) and billing processing delays for ~1,000 bills (Calltower). Root cause of the early 500-error spike.
🚀
Recommended: Client Comms Before April
Auth service rewrite completes early April. Proactive communication to affected clients (Novalera, Truline, Premise, Raven Tech, Newport) before and after release will reduce call volume and rebuild trust.

🎯 If You Fix One Thing: Self-Service Password Reset

Password reset issues appear in 159 of 2,637 calls — more than any other single issue. Each one requires SA or admin time. Fixing the front-end reset flow would be the single highest-impact action for reducing support burden and client friction across the board.
